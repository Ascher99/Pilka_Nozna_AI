                    SPIS TREŚCI
          Projekt i implementacja inteligentnej platformy predykcyjnej

          porównawczych modeli uczenia maszynowego

ROZDZIAŁ 1: WSTĘP
────────────────────────────────────────────────────────────────────────────────

1.1. Wprowadzenie do tematyki i motywacja
   Opis problematyki przewidywania zdarzeń losowych w sporcie. Motywacja wynikająca
   z rosnącego rynku analityki sportowej (Sports Analytics) oraz wyzwań
   związanych z modelowaniem danych stochastycznych.

1.2. Cel i teza pracy
   Celem jest opracowanie kompletnego systemu informatycznego (full-stack) oraz
   przeprowadzenie eksperymentu badawczego porównującego skuteczność modeli
   liniowych (Regresja Logistyczna) z modelami zespołowymi (Random Forest)
   w zadaniu klasyfikacji wieloklasowej wyników meczów.
   Teza: "Zastosowanie złożonych modeli zespołowych (Ensemble Methods) pozwala na
   uzyskanie wyższej jakości predykcji probabilistycznej (Log Loss) w porównaniu
   do klasycznych metod statystycznych w piłce nożnej".

1.3. Zakres pracy
   System obejmuje dwie ligi (Premier League, Ekstraklasa), zakres danych 2021-2025.
   Praca pokrywa cały cykl życia oprogramowania: od pozyskania danych (ETL),
   przez trening modeli (ML), po udostępnienie wyników przez REST API i
   interfejs webowy.

1.4. Struktura pracy
   Krótki opis zawartości kolejnych rozdziałów.

ROZDZIAŁ 2: ANALIZA DZIEDZINOWA I PRZEGLĄD LITERATURY
────────────────────────────────────────────────────────────────────────────────

2.1. Specyfika danych w piłce nożnej
   Analiza czynników wpływających na wynik: forma, atut własnego boiska (Home Advantage),
   losowość. Omówienie problemu niezbalansowania klas (częste zwycięstwa gospodarzy).

2.2. Istniejące rozwiązania i ich ograniczenia
   Przegląd serwisów komercyjnych (SofaScore, Flashscore) oraz rozwiązań akademickich.
   Identyfikacja luki: brak otwartych systemów łączących przejrzystość modelu
   (Explainable AI) z nowoczesnym interfejsem użytkownika.

2.3. Metryki oceny w predykcji sportowej
   Dlaczego samo "Accuracy" nie wystarcza? Wprowadzenie pojęć:
   - Log Loss (ocena pewności modelu)
   - F1-Macro (ocena zbalansowana dla remisów)
   - Calibration (wiarygodność prawdopodobieństw)

ROZDZIAŁ 3: METODOLOGIA BADAŃ I TEORIA ALGORYTMÓW
────────────────────────────────────────────────────────────────────────────────

3.1. Definicja problemu uczenia maszynowego
   Sformułowanie problemu jako klasyfikacji wieloklasowej nadzorowanej (Supervised
   Multi-class Classification).

3.2. Wybór i charakterystyka algorytmów
   3.2.1. Model Referencyjny (Baseline): Regresja Logistyczna
          Matematyczne podstawy, funkcja Softmax, zalety (interpretowalność).
   3.2.2. Model Docelowy (Challenger): Random Forest
          Koncepcja uczenia zespołowego (Bagging), drzewa decyzyjne,
          odporność na nieliniowości i interakcje między cechami.

3.3. Inżynieria cech (Feature Engineering)
   Opis procesu tworzenia zmiennych objaśniających:
   - Agregacja formy z okien czasowych (Rolling Window)
   - Kodowanie zmiennych kategorycznych (Label Encoding)
   - Skalowanie cech numerycznych (StandardScaler)

3.4. Strategia walidacji modelu
   Krytyczna analiza metod walidacji. Uzasadnienie użycia walidacji chronologicznej
   (Time Series Split) zamiast losowej (K-Fold) w celu uniknięcia wycieku danych
   (Data Leakage) z przyszłości.

ROZDZIAŁ 4: PROJEKT SYSTEMU I ARCHITEKTURA
────────────────────────────────────────────────────────────────────────────────

4.1. Wymagania funkcjonalne i niefunkcjonalne
   Zdefiniowanie wymagań: czas odpowiedzi API < 200ms, skalowalność na nowe ligi,
   responsywność interfejsu (RWD), separacja warstw.

4.2. Architektura logiczna systemu
   Opis architektury klient-serwer z wykorzystaniem REST API.
   Diagram komponentów:
   [Źródła Danych] -> [Moduł ETL] -> [Baza Danych/Pliki] -> [API Predykcyjne] -> [Frontend]

4.3. Projekt przepływu danych (Data Flow)
   Szczegółowy opis potoku przetwarzania danych (Pipeline):
   Pobranie (Extract) -> Czyszczenie i Transformacja (Transform) -> Trening -> Ewaluacja -> Serwowanie.

4.4. Stos technologiczny
   Uzasadnienie doboru technologii:
   - Backend: Python, FastAPI (wydajność, asynchroniczność)
   - ML: Scikit-learn, Pandas, Joblib
   - Frontend: Next.js, React, TypeScript (SSR, bezpieczeństwo typów)
   - Konteneryzacja (opcjonalnie): Docker

ROZDZIAŁ 5: IMPLEMENTACJA SYSTEMU
────────────────────────────────────────────────────────────────────────────────

5.1. Implementacja warstwy danych i ETL
   Omówienie skryptów pobierających i normalizujących dane.
   Rozwiązanie problemu niespójnych nazw drużyn (Mapowanie encji).
   Struktura przechowywania danych treningowych (CSV vs Database).

5.2. Implementacja modułu uczenia maszynowego
   Opis klasy trenującej. Implementacja eksperymentu porównawczego w Pythonie.
   Zarządzanie cyklem życia modelu (zapisywanie artefaktów .pkl, wersjonowanie).

5.3. Implementacja API Backendowego (FastAPI)
   Budowa punktów końcowych (endpoints). Modele danych Pydantic (walidacja wejścia/wyjścia).
   Obsługa błędów i wstrzykiwanie zależności (Dependency Injection) dla modeli ML.

5.4. Implementacja Interfejsu Użytkownika (Frontend)
   Struktura komponentów React. Wizualizacja formy (Form Bar).
   Integracja z API i zarządzanie stanem aplikacji.
   Konfiguracja środowiska (zmienne .env).

ROZDZIAŁ 6: WYNIKI BADAŃ I EWALUACJA SYSTEMU
────────────────────────────────────────────────────────────────────────────────

6.1. Przebieg eksperymentu
   Opis środowiska testowego i parametrów treningu.

6.2. Analiza porównawcza modeli (Wyniki ML)
   Prezentacja wyników w tabelach (Baseline vs Challenger).
   - Porównanie Accuracy i F1-Score dla obu lig.
   - Analiza Log Loss – który model jest "pewniejszy"?
   - Macierz pomyłek (Confusion Matrix) – analiza błędów dla klasy "Remis".

6.3. Analiza ważności cech (Feature Importance)
   Weryfikacja hipotez: czy ostatnie mecze faktycznie wpływają na wynik?
   Które cechy (gole vs punkty) mają większą wagę decyzyjną?

6.4. Testy systemowe i wydajnościowe
   - Testy jednostkowe logiki biznesowej.
   - Testy integracyjne API (scenariusze sukcesu i błędów).
   - Manualne testy akceptacyjne interfejsu użytkownika.

6.5. Dyskusja wyników
   Interpretacja rezultatów w kontekście postawionej tezy.
   Dlaczego model X zadziałał lepiej/gorzej? Wpływ losowości sportu na maksymalną
   osiągalną dokładność.
